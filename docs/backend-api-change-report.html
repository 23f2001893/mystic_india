Mystic India Change Report

Date: June 18, 2026

Summary

The project was moved from static story data in src/data/stories.js to dynamic FastAPI + PostgreSQL data, then extended with authentication, separate admin story/category CRUD pages, publish controls, category-filtered story management, and an admin analytics dashboard.

Backend Changes

backend/app.py
  • Added FastAPI metadata.
  • Added CORS for the Vite dev server at localhost:5173.
  • Kept SQLAlchemy table creation on startup.
backend/config.py
  • Added DATABASE_URL environment variable support.
  • Kept the local PostgreSQL URL as the default fallback.
backend/models.py
  • Cleaned SQLAlchemy model formatting.
  • Added unique indexed slugs for categories and stories.
  • Used existing User, Categories, and Stories tables.
backend/schemas.py
  • Added Pydantic response schemas for categories and stories.
  • Added request/response schemas for story create/update.
  • Added request/response schemas for category create/update.
  • Added request/response schemas for registration and login.
backend/controllers.py
  • Replaced placeholder story endpoint with real database-backed APIs.
  • Added story list, detail, related, featured, and search endpoints.
  • Added password hashing and signed bearer-token authentication.
  • Added admin-only create, update, and delete story endpoints.
  • Added admin-only create, update, and delete category endpoints.
  • Protected category deletion when stories still belong to that category.
  • The first registered user becomes admin; later users become user.

API Endpoints Added

  • POST /api/auth/register - register user and return token.
  • POST /api/auth/login - login user and return token.
  • GET /api/auth/me - return current authenticated user.
  • GET /api/categories - return categories with story counts.
  • GET /api/stories?category=all&sort=popular - return database stories.
  • GET /api/stories/featured?limit=3 - return top stories by popularity.
  • GET /api/stories/{slug} - return one story by slug.
  • GET /api/stories/{story_id}/related?limit=3 - return related stories.
  • GET /api/search/stories?q=ram - search title, description, and category.
  • POST /api/admin/categories - admin creates a category.
  • PUT /api/admin/categories/{category_id} - admin updates a category.
  • DELETE /api/admin/categories/{category_id} - admin deletes an empty category.
  • POST /api/admin/stories - admin creates a story.
  • PUT /api/admin/stories/{story_id} - admin updates a story.
  • DELETE /api/admin/stories/{story_id} - admin deletes a story.

Rollback Options

Recommended: Make a commit before experimenting further, so rollback is easy.
git add .
git commit -m "Add backend story APIs auth and admin CRUD"
  • To inspect changed files: git status --short
  • To discard all uncommitted changes and return to the previous checked-out version: git restore .
  • To remove newly created untracked files after restoring: git clean -fd
  • To go back after committing: git revert <commit-hash>
Careful: git restore . and git clean -fd can remove local work. Use them only when you are sure you do not need the current edits.
